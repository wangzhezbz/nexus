# 怎么把 @nexus/ui、@nexus/auth 发给各团队装

现在两个包在本仓库 `platform/nexus-ui`、`platform/nexus-auth`，**还没发布**。各团队要能 `pnpm add`，二选一：

## 方案 A：Git 安装（最省事，推荐先用）
1. 把这两个包各推到一个 git 仓库（或同一个仓库的两个目录）。最简单：把整个 `platform/` 推到一个私有 git 仓库 `nexus-shared`。
2. 团队安装（pnpm 支持子目录）：
   ```bash
   pnpm add "git+https://你的git地址/nexus-shared.git#path:/nexus-ui"
   pnpm add "git+https://你的git地址/nexus-shared.git#path:/nexus-auth"
   ```
3. 更新：改完包 push 后，团队 `pnpm update @nexus/ui @nexus/auth`。
> 因为包是直接发 TS 源码，团队 `next.config` 必须 `transpilePackages:['@nexus/ui','@nexus/auth']`（已写进对接通知）。

## 方案 B：私有 npm（规模大了再上）
1. 搭私有 registry（Verdaccio 最简单，一条 `npx verdaccio`），或用 GitHub Packages。
2. 两个包 `npm publish`（先 `npm config set registry 你的私有源`）。
3. 团队照常 `pnpm add @nexus/ui @nexus/auth`。

## 你现在要做的三件
1. 选 A 或 B，把两个包放出去，把安装地址发群里。
2. 把 `platform/.env` 里的 **JWT_SECRET** 私下发给各后端团队（已生成，勿入库）。
3. 服务器上 `cd platform && cp .env.example .env`（填密码、把 JWT_SECRET 填成同一个）→ `docker compose up -d` 起中央认证 → 配 nginx（`platform/nginx/shanhaiyouling.conf`）。

完成后各团队照《对接通知.md》接即可。
